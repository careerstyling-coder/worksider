# 01-prd.md: 제품 요구사항 명세 (PRD)

**프로젝트**: Workside (워크사이드)
**버전**: 1.0
**작성일**: 2026-03-16
**상태**: MVP Phase 1 기획

---

## 1. 제품 개요

### 1.1 한 줄 설명
일을 더욱 일답게 하고자 하는 직장인들의 성장 커뮤니티 플랫폼

### 1.2 Cogito Statement (개발자의 의도)
나는 일에 대한 태도의 '다름'을 겉으로 꺼내는 언어를 만들고 싶다. 왜냐하면 사람을 이해하는 것은 불가능하지만, 일에 대한 태도의 다름은 볼 수 있고, 그것이 보이면 함께 일하는 법을 찾을 수 있기 때문이다.

### 1.3 Eros Idea (공동 욕구)
사람은 자기 자신을 이해하고 싶어 하고, 그 이해를 타인과 나누고 싶어 한다. 이 욕구가 특정 맥락에 구체화될 때 가치가 생기고, 공유 가능한 언어가 만들어질 때 커뮤니티가 형성된다.

## 2. 사용자 세그먼트

| 세그먼트 | 설명 | 주요 행동 |
|---------|------|---------|
| **일반 직장인 (Primary)** | 일을 더 잘하고 싶은 개인 | DNA 진단 → 결과 공유 → 재진단 또는 이탈 |
| **성장 참여자 (Secondary)** | 정기적으로 질문에 답하고 다른 유형의 시각을 탐색 | 질문 참여 → 결과 분석 → 제안 → Shout out |
| **플랫폼 관리자 (Admin)** | 질문 등록, 제안 채택, 통계 관리 | 대시보드 모니터링 → 질문 배포 → 제안 채택 |

## 3. 핵심 기능 (MVP Phase 1)

### FEAT-1: Work Style DNA 진단
**목표**: 공통 언어 제공을 통한 자기 이해 플랫폼 구축

**기능 명세**:
- **진단 방식**: 4축(P·C·Pol·S) 기반 비정규화 스코어링
  - P (실무 Practice): 업무 전문성과 실행력
  - C (관계 Communication): 팀원 협력과 관계 형성
  - Pol (정치 Politics): 조직 내 영향력과 의사 결정 참여
  - S (자율 Self-leadership): 자율성과 독립적 주도성
- **세미 버전**: 12문항 (축당 3문항 고정, 축 내 랜덤) / 3~5분
- **풀 버전**: 40문항 (축당 10문항: 행동 4 + 상황 4 + 가치 2) / 15~20분
- **스코어링**: 각 축 0~100 정규화
  - 행동형: Likert 1~7
  - 상황형/가치형: L=1, M=4, H=7
- **비회원 접근**: 링크 하나로 바로 시작 가능 (가입 없이 진단 및 결과 열람 허용)
- **결과 레이더 차트**: 시각적 4축 표현 + 축별 정성 묘사
- **참조 페르소나**:
  - 실무형 전문가 (P 높음)
  - 조직형 정치인 (Pol 높음)
  - 관계형 촉매자 (C 높음)
  - 자율형 독립가 (S 높음)
  - 전략적 성과자 (P+Pol 높음)
  - 안정적 조력자 (C+S 높음)

**왜?**: 공통 언어를 만들려면 먼저 개인의 일 태도를 구조화해야 한다. 진단 결과가 '나는 어떤 사람인가'의 언어가 된다.

**우선순위**: MVP 필수

**완료 조건**:
- 세미/풀 버전 모두 배포됨
- 비회원 진단 → 결과 흐름 작동
- 레이더 차트 정확하게 렌더링
- 결과 화면에서 4축 묘사 텍스트 표시

---

### FEAT-2: 공유 (SNS/카카오/링크)
**목표**: 바이럴 확산과 사용자 획득 가속화

**기능 명세**:
- **공유 채널**:
  - Twitter/X (OG 카드 자동 생성)
  - Facebook (OG 카드 자동 생성)
  - Kakao Talk (커스텀 메시지 + 썸네일)
  - 직접 링크 복사 (공유 URL: `/{shareId}`)
- **공유 링크 수신 화면** (`/share/:shareId`):
  - 공유자의 DNA 결과를 먼저 노출 (이름 없이, 점수와 페르소나만)
  - '나는 어떨까?' CTA 버튼 → DNA 진단 시작
  - 수신자 진단 후 비교 뷰 (구현은 Phase 2)
- **OG 이미지 동적 생성**:
  - Next.js @vercel/og 활용
  - 개인의 4축 스코어 + 페르소나 이름을 카드 형태로 표시
  - 예: "내 일의 스타일은? P:72 C:45 Pol:38 S:61 → 전략적 성과자"

**왜?**: '다름의 언어'를 퍼뜨리려면 공유가 핵심. MBTI처럼 '나는 P=82' 라벨이 SNS에서 정체성 표현 도구가 됨.

**우선순위**: MVP 필수

**완료 조건**:
- SNS 공유 버튼 (Twitter/Facebook/Kakao) 클릭 시 정확한 OG 카드 생성
- 공유 링크 수신 화면 정상 로드
- '나는 어떨까?' CTA 클릭 시 진단 흐름 시작

### FEAT-3: 질문 참여 (피드)
**목표**: 정기적 재방문 유도 및 커뮤니티 데이터 축적

**기능 명세**:
- **질문 유형**:
  - 단순 1회성: 하나의 선택지 선택 (예: "당신의 업무 스타일은?")
  - 설문형 다문항: 여러 개 질문 연쇄 제시 (예: "일에 대한 태도 3가지")
- **참여 메커니즘**:
  - 비회원: 익명 응답만 제출, 결과 열람 불가 (가입 유도)
  - 회원: 익명 응답 + 자신의 응답 이력 저장 + 결과 열람 가능
- **접근 제어**:
  - 진행 중인 질문만 응답 가능
  - 마감된 질문은 결과만 열람 (반응 불가)
  - 내가 참여한 질문: 특별 마킹으로 구분
- **참여 결과 잠금 해제**:
  - 미참여 시 결과 화면에 "참여하여 결과를 확인하세요" 메시지
  - 참여 후 즉시 결과 열람
- **사용자 제안 메커니즘**:
  - 누구나 질문 제안 가능 (`/suggest` 화면)
  - 제안에는 주제 + 배경 설명 필수
  - 관리자가 채택/반려 → 채택 시 제안자에게 알림
  - 채택된 제안은 질문으로 변환되어 배포
- **Shout out 반응**:
  - 제안 항목에 하트/응원 반응 추가
  - 반응 개수는 채택 가능성 신호로 활용
- **피드 통합 뷰** (`/feed`):
  - 채택된 질문 상단 고정
  - 질문 목록 필터: 진행중 / 마감 / 내 참여 상태
  - 제안 목록 (최신순 또는 Shout out 순)
  - 단일 피드로 통합 (개별 페이지 아님)

**왜?**: 재방문의 핵심 엔진. '참여에 따른 보상(결과 보기)'이 서비스 철학. 참여할수록 데이터 신뢰도 상승 → 참여 동기.

**우선순위**: MVP 필수

**완료 조건**:
- 질문 목록 렌더링 (진행중/마감 표시)
- 회원 응답 제출 및 저장
- 비회원 응답 제출 (결과 미노출, 가입 유도)
- 참여 후 결과 화면 열람 가능
- 질문 제안 폼 제출 + 관리자 검토 워크플로우

## 5. 네비게이션 플로우

### 5.1 공유 링크 수신 플로우
공유 링크 → 공유 수신 화면 → DNA 진단 → DNA 결과 → **가입 게이트** → 피드

### 5.2 직접 방문 플로우
검색/직접 방문 → 랜딩 페이지 → DNA 진단 → DNA 결과 → **가입 게이트** → 피드

### 5.3 로그인 사용자 플로우
로그인 → 피드(메인) ↔ 내 DNA / 질문 참여 / 세부 결과 / 질문 제안

### 5.4 알림 플로우
알림(이메일) → 질문 직접 이동 또는 피드 경유

## 7. MVP 범위 정의

### 포함 사항 (Phase 1)
- 세미 + 풀 DNA 진단 (비회원 접근 가능)
- DNA 결과 레이더 차트 + 축별 설명
- SNS/카카오/링크 공유 + OG 이미지 동적 생성
- 공유 링크 수신 화면
- 회원가입 (이메일 + 닉네임 + 산업군 + 회사 규모)
- 질문 참여 (단순/다문항) + 익명 응답
- 질문 결과 (집계 %, 유형별 분포, 소수 의견)
- 질문 제안 폼 + 관리자 승인 워크플로우
- 피드 통합 뷰 (질문 + 제안 목록)
- 기본 관리자 대시보드 (지표 열람, 질문 배포)

### 제외 사항 (Phase 2+)
- 비교 기능 (공유자 vs 수신자 DNA 비교)
- 결과 심화 분석 (심화 PDF 리포트)
- 코칭 세션 연결
- 실시간 알림 (Realtime Supabase)
- 커뮤니티 피드 (다른 사용자 결과 공개 공유)
- 고급 분석 (시계열 진단 비교)

## 8. 성공 지표 (KPI)

### North Star Metric
**DNA 진단 완료 → 질문 재참여율**

질문 참여 후 다음 질문에도 돌아오는 사용자의 비율

### 핵심 지표

| 지표 | 목표 (MVP 엔드) | 측정 방식 |
|-----|-----------------|----------|
| DNA 진단 완료 수 | 100명+ | Supabase 쿼리 |
| 질문 재참여율 | 30%+ | (2차 참여자) / (1차 참여자) |
| 공유 클릭 CTR | 5%+ | 공유 버튼 클릭 / 결과 노출 |
| 회원가입 전환율 | 10%+ | 가입 완료 / 진단 완료 |
| 질문 제안 수 | 10+ | 관리자 검토 대기 중인 제안 |
| 일일 활성 사용자 (DAU) | 10+ | 어떤 행동이든 한 번이라도 한 사용자 |

### 게이트 기준 (Phase 1 → Phase 2)
1. DNA 진단 완료 수 100명 이상 도달
2. 질문 재참여율 30% 이상 달성 (한 번 온 사람이 다음 질문에도 참여)
3. 자발적 공유 행동 확인 (공유 버튼 클릭 5회 이상)

## 9. 가설 vs 사실 (Assumption vs Validated)

### 가설 (미검증)

| ID | 가설 | 검증 방법 | 기대값 |
|----|------|---------|--------|
| H-1 | 일반 직장인이 '업무 스타일 진단'에 참여 의향이 있다 | DNA 진단 완료 수 > 50 | 3주 내 100명 |
| H-2 | '참여해야 결과를 본다' 구조가 재방문을 유도한다 | 질문 재참여율 > 20% | 첫 재참여 주기에서 >30% |
| H-3 | SNS 공유가 신규 사용자 유입의 주요 채널이다 | 공유 링크 유입 > 전체 30% | 2주 내 확인 |
| H-4 | MBTI처럼 직장인 맥락에서도 바이럴될 수 있다 | 공유 클릭 CTR > 5% | 1주 내 확인 |

### 사실 (검증됨)
- MVP Phase 1 구현 가능 (1인 + AI 보조)
- Next.js + Supabase로 3~4주 내 배포 가능

## 10. 실패 시나리오 & 대응

| 시나리오 | 신호 | 대응 |
|---------|------|------|
| 진단 완료 수 정체 (<20명 @ 2주) | 낮은 인지 + 낮은 진입 의욕 | 랜딩 페이지 UX 재검토, 설명 텍스트 명확화 |
| 재참여율 0% (2주차 질문 참여 0) | 일회성 사용자화, '참여해야 결과' 구조 실패 | 질문 주기 단축, 이메일 알림 도입 |
| 공유율 0% (공유 버튼 클릭 0) | 바이럴 채널 실패 | OG 이미지 개선, SNS 콘텐츠 마케팅 |
| 가입 전환율 <5% | 회원 제한 기능 부족 or 진단만으로 만족 | 문구 조정, 필요 기능(비교) 전면 배치 |

## 11. 타협 기록 (Tradeoff)

| 기능 | 결정 | 이유 |
|------|------|------|
| 비교 기능 (공유자 vs 수신자) | Phase 2로 연기 | MVP 범위 축소 + 구현 복잡도 높음 |
| 실시간 알림 | Phase 2로 연기 | Supabase Realtime 학습곡선 + 우선순위 낮음 |
| 심화 PDF 리포트 (유료) | Phase 2로 연기 | 수익화 전 사용자 신뢰 구축 우선 |
| 커뮤니티 공개 피드 | Phase 2로 연기 | 익명 구조와 충돌 가능성 + 모더이션 복잡도 |
| 가입 게이트 고정 | 모듈화로 유연화 | A/B 테스트를 위해 관리자 설정 가능하게 구현 |

## 12. 벤치마크 분석

### 경쟁자 비교

| 이름 | 카테고리 | 유사점 | 차이점 | 배울점 |
|------|---------|--------|--------|--------|
| 16Personalities (MBTI) | 성격 진단 + 바이럴 | 자기 정의 + SNS 공유 | 성격 전반, 일 특화 아님 | 바이럴 메커니즘 (라벨링 + 공유) |
| StrengthsFinder | 일 역량 진단 | 일 중심 진단 | B2B 유료 모델, 개인 접근 어려움 | 신뢰도 높은 진단 설계 |
| DISC | 행동 스타일 진단 | 4축 기반 분류 | 기업 교육 중심, 개인 접근 거의 없음 | 다양한 상황형 문항 활용 |
| 잡플래닛/블라인드 | 직장인 커뮤니티 | 익명성 + 직장인 대상 | 회사 리뷰/익명 소통, 진단 없음 | 커뮤니티 활성화 메커니즘 |

### 시장 공백 (Workside의 위치)
**일 특화 + 바이럴** = 빈 자리

- MBTI: 바이럴 강함 + 일 특화 약함
- StrengthsFinder/DISC: 일 특화 강함 + 바이럴 약함
- **Workside: 일 특화 + 바이럴 = 기회**

## 16. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| DNA | "Work Style DNA" - 4축 기반 업무 스타일 진단 결과 |
| 세미 버전 | 12문항, 3~5분 진단 (빠른 버전) |
| 풀 버전 | 40문항, 15~20분 진단 (상세 버전) |
| 페르소나 | DNA 결과로 도출된 6가지 직장인 유형 분류 |
| 공유 링크 | 진단자가 결과를 SNS에 공유했을 때의 고유 URL |
| 익명 응답 | 사용자 신원 노출 없이 질문에 응답 |
| 소수 의견 (Minority View) | 특정 유형이 다수와 다르게 선택한 항목 강조 |
| 재참여율 | 1회 참여자 중 2회차 질문에도 참여한 비율 |
| 가입 게이트 | 진단/공유 후 회원가입을 강제하는 위치 (모듈화 가능) |
