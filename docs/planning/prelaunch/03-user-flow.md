# 03-user-flow.md: 사용자 흐름도

**프로젝트**: Workside 사전 예약 페이지 (프리론칭)
**버전**: 1.0
**작성일**: 2026-03-25
**상태**: 사용자 시나리오 설계 완료

---

## MVP 캡슐 (10줄)

5가지 메인 플로우: (1) 신규 방문 → 예약, (2) 예약 → 초대, (3) 초대 링크 수신 → 예약, (4) 재방문 → 대시보드, (5) 관리자 → 통계 확인. 각 플로우는 성공/실패/엣지 케이스 포함. 성공 경로: 공감 메시지 → DNA 설명 → 예약폼 → 예약완료 (순번+초대링크). 초대 경로: 초대 링크 클릭 → 개인화 배너 → 예약 → 초대자 진행바 업데이트. 엣지 케이스: 이미 예약한 이메일 재시도, 자기 초대 링크로 접속, 초대자 정보 조회 불가. 모든 경로는 모바일 우선 고려.

---

## 1. 메인 플로우 (5가지)

### Flow 1: 신규 방문 → 예약 (Happy Path)

```
사용자 → workside.day/prelaunch 접속
         ↓
         랜딩 페이지 로드 (공감 메시지 노출)
         ↓
         [HeroSection 확인]
         - 제목: "당신의 일 스타일을 먼저 알아보세요"
         - 부제: "첫 500명을 위한 특별한 기회"
         ↓
         스크롤하여 DNA 소개 확인
         ↓
         [DNAIntroSection 확인]
         - 4축 설명 (P/C/Pol/S)
         - 페르소나 6가지 예시
         ↓
         예약 폼 보임
         ↓
         [ReservationForm 입력]
         - 이메일 입력: "user@example.com"
         - 직군 선택: "IT" (드롭다운)
         - 연차 선택: "4~5년차" (드롭다운)
         ↓
         "지금 예약하기" 버튼 클릭
         ↓
         [API: POST /api/prelaunch/reserve]
         - 이메일 중복 검사: OK
         - 초대코드 생성: "V1StGXR_Z5j3eK5t2nK6"
         - DB 저장 완료
         ↓
         예약 완료 화면으로 이동 (/prelaunch/reserved)
         ↓
         [ReservedPage 노출]
         - "예약되었습니다!" 메시지
         - 순번 표시: "#123"
         - 초대 링크 카드 표시
         - 초대 진행바: 0/5
         - 리워드 안내 표시
         - SNS 공유 버튼 표시
         ↓
         사용자 경험 완료 ✓
```

**성공 조건**:
- 폼 제출 성공
- 초대코드 생성 및 저장
- 예약 완료 화면 로드

---

### Flow 2: 예약 → 초대 (Invite Share)

```
사용자 (이미 예약함) → /prelaunch/reserved 페이지 방문
                    ↓
                    초대 링크 카드 확인
                    ↓
                    [선택 A: 링크 복사]
                    "링크 복사" 버튼 클릭
                    ↓
                    클립보드에 복사: "https://workside.day/?ref=V1StGXR_Z5j3eK5t2nK6"
                    ↓
                    사용자가 카카오톡/이메일로 붙여넣기
                    ↓
                    OR
                    ↓
                    [선택 B: SNS 공유]
                    "카카오톡 공유" 버튼 클릭
                    ↓
                    [API: POST /api/prelaunch/invite/track]
                    - invite_code: "V1StGXR_Z5j3eK5t2nK6"
                    - 공유 이벤트 기록
                    ↓
                    카카오톡 앱으로 이동
                    ↓
                    채팅방 선택 후 메시지 전송
                    ↓
                    초대 완료 ✓
```

**성공 조건**:
- 링크 복사 또는 SNS 공유 버튼 클릭
- 공유 이벤트 DB 기록

---

### Flow 3: 초대 링크 수신 → 예약 (Invitee Path)

```
사용자 (초대받음) → 카카오톡/이메일/SNS에서 초대 링크 클릭
                 ↓
                 workside.day/?ref=V1StGXR_Z5j3eK5t2nK6 접속
                 ↓
                 [API: POST /api/prelaunch/invite/track]
                 - ?ref 파라미터 인식
                 - invite_tracking 테이블에 link_clicked = true 기록
                 ↓
                 [페이지 렌더링]
                 - 초대 배너 표시: "OO님이 초대했어요 💌"
                 - (초대자 정보는 익명 처리 또는 이메일만)
                 ↓
                 랜딩 페이지 콘텐츠 표시 (Flow 1과 동일)
                 ↓
                 예약 폼 입력
                 ↓
                 "지금 예약하기" 버튼 클릭
                 ↓
                 [API: POST /api/prelaunch/reserve]
                 - 요청 파라미터: ?ref=V1StGXR_Z5j3eK5t2nK6 포함
                 - 초대코드 생성 및 invited_by_id 설정
                 ↓
                 [DB 업데이트]
                 - reservations 테이블: invited_by_id = 초대자 ID
                 - invite_tracking: converted = true, converted_at = NOW()
                 - rewards: 초대자의 invite_success_count += 1
                 ↓
                 예약 완료 화면 로드
                 ↓
                 초대자의 대시보드 (다른 탭에서 열려있으면) 실시간 업데이트
                 ↓
                 초대 전환 완료 ✓
```

**성공 조건**:
- ?ref 파라미터 감지
- link_clicked = true 기록
- 예약 후 converted = true 기록
- 초대자 카운트 증가

---

### Flow 4: 재방문 → 대시보드 (User Returning)

```
사용자 (이미 예약함) → workside.day 접속
                    ↓
                    로그인되어 있는가?
                    ├─ YES → /prelaunch/my-reservation으로 자동 리다이렉트
                    └─ NO  → 홈페이지로 표시 (예약 여부 기억 안 함)
                    ↓
                    [MyReservationPage 로드]
                    ↓
                    표시 정보:
                    - "내 순번: #123"
                    - "초대 현황: 1/5 (1명 초대됨)"
                    - "리워드 상태: 4명 더 초대하면 배지 획득"
                    - "초대 링크: [복사] [공유]"
                    ↓
                    사용자가 추가 초대를 위해 링크 공유
                    ↓
                    Flow 2와 동일 (SNS 공유)
                    ↓
                    또는
                    ↓
                    다시 돌아와서 진행률 확인
                    ↓
                    초대자 페이지 refresh → 업데이트된 카운트 표시
                    ↓
                    4주 후 본 서비스 오픈 시 얼리어답터 배지 자동 적용
```

**성공 조건**:
- 로그인 상태 유지
- 초대 현황 실시간 반영
- 추가 초대 가능

---

### Flow 5: 관리자 → 통계 확인 (Admin Dashboard)

```
관리자 → /admin 페이지 접속
        ↓
        사이드바 또는 탭에서 "사전 예약" 선택
        ↓
        /admin/prelaunch-dashboard 페이지 로드
        ↓
        [API: GET /api/admin/prelaunch/dashboard]
        ↓
        대시보드 지표 표시:
        ├─ 총 예약 수: 450명
        ├─ 초대 전환율: 18.5%
        ├─ 얼리어답터 배지 획득: 87명
        ├─ 평균 초대자당 성공 초대: 2.1명
        ├─ 직군 분포: IT(180), Finance(120), ...
        └─ 일별 예약 추이: 꺾은선 차트
        ↓
        관리자가 지표를 분석하여 A/B 테스트 결정
        ├─ 초대 전환율 < 15%? → 리워드 메시지 변경
        ├─ 특정 직군 저조? → 타겟팅 조정
        └─ DAU 감소? → 알림 추가
        ↓
        필터 적용 (기간 선택)
        ↓
        대시보드 데이터 재로드
        ↓
        CSV 내보내기 (선택)
        ↓
        통계 분석 완료 ✓
```

**성공 조건**:
- 대시보드 지표 정확하게 표시
- 필터 적용 작동
- 5분마다 자동 갱신

## 2. 엣지 케이스 (Edge Cases)

### Case 1: 이미 예약한 이메일 재시도

```
사용자 → 이미 예약한 이메일로 다시 폼 제출
        ↓
        [API: POST /api/prelaunch/reserve]
        ↓
        에러: email already exists (UNIQUE 제약 위반)
        ↓
        프론트엔드: "이미 예약하신 이메일입니다. 로그인하세요" 메시지
        ↓
        [버튼 옵션]
        - "내 예약 확인하기" → /prelaunch/my-reservation
        - "다른 이메일로 예약" → 폼 초기화
        ↓
        사용자 선택
```

**처리**:
- DB 에러 → 클라이언트 에러 메시지 반환
- 에러 상태 관리: `useReservationForm` 훅에서 처리

---

### Case 2: 자기 초대 링크로 접속

```
사용자 → 자신이 생성한 초대 링크 클릭 (?ref=자신의코드)
        ↓
        [API: POST /api/prelaunch/invite/track]
        ↓
        link_clicked = true 기록
        ↓
        랜딩 페이지 표시 (초대 배너 자신을 표시 또는 숨김)
        ↓
        예약 폼 표시
        ↓
        이미 예약한 이메일로 제출 시도
        ↓
        에러: "이미 예약하신 이메일입니다"
        ↓
        사용자는 추가 예약 불가능 (의도된 동작)
```

**처리**:
- 자기 초대는 카운트하지 않음 (invited_by_id = NULL)
- link_clicked만 증가 (converted는 증가 안 함)

---

### Case 3: 초대 링크 만료 (선택)

```
사용자 → 4주 후 초대 링크 클릭
        ↓
        캠페인 종료 후 링크 비활성화되었는가?
        ├─ YES (선택) → "캠페인이 종료되었습니다" 메시지
        └─ NO (대부분) → 계속 작동 (4주 후에도 가입 가능)
        ↓
        대부분의 경우 계속 작동 (사전예약과 본서비스 연결)
```

**처리**:
- Phase 1에서는 링크 만료 없음
- Phase 2에서 필요시 추가

---

### Case 4: 초대자 정보 조회 (개인정보)

```
사용자 → "누가 나를 초대했는가?" 궁금함
        ↓
        invited_by_id 필드로 초대자 특정 가능 (DB에만)
        ↓
        초대자 이메일/이름 노출할 것인가?
        ├─ YES → 초대자 이메일 표시 (선택, Phase 2)
        └─ NO  → 초대자 정보 비공개 (현재, Phase 1)
        ↓
        현재: 초대 배너에 "OO님이 초대했어요" (익명)
        ↓
        4주 후 본 서비스 오픈 시 이름 노출 (선택)
```

**처리**:
- Phase 1: 초대자 이메일 미노출 (privacy 중시)
- Phase 2: 초대자 미니 프로필 노출 (관계 형성)

---

### Case 5: 네트워크 오류 중 예약 (Offline)

```
사용자 → 예약 폼 입력 중 인터넷 끊김
        ↓
        "지금 예약하기" 버튼 클릭
        ↓
        [API 요청 타임아웃 (예: 10초)]
        ↓
        에러 처리:
        - "네트워크 연결을 확인해주세요" 메시지
        - 재시도 버튼 표시
        ↓
        사용자가 재시도 클릭
        ↓
        API 재요청 (동일한 이메일)
        ↓
        성공 또는 "이미 예약됨" 에러
```

**처리**:
- 타임아웃: 10초
- 재시도 로직: 자동 또는 수동 버튼
- Idempotency: 같은 이메일은 중복 예약 안 됨 (UNIQUE)

---

### Case 6: Rate Limiting (스팸 방지)

```
공격자 → IP별로 예약 API 대량 호출
        ↓
        [Rate Limiter 작동]
        - 1분당 10회 초과 시 429 Too Many Requests
        ↓
        응답: "잠시 후 다시 시도해주세요"
        ↓
        1분 경과 후 재시도 가능
```

**처리**:
- 미들웨어: IP별 rate limiting
- Threshold: 10회/분
- 응답: 429 상태코드 + 재시도 안내

## 4. 오류 상태 (Error States)

### 유효성 검사 오류

| 필드 | 유효하지 않음 | 에러 메시지 |
|------|-------------|----------|
| 이메일 | "test" (@ 없음) | "유효한 이메일 형식을 입력하세요" |
| 이메일 | "test@example.com" (중복) | "이미 예약하신 이메일입니다" |
| 직군 | (선택 안 함) | "직군을 선택하세요" |
| 연차 | (선택 안 함) | "연차를 선택하세요" |

### 서버 오류

| 상황 | 상태코드 | 메시지 |
|------|---------|--------|
| 이메일 중복 | 409 Conflict | "이미 예약되었습니다" |
| Rate limit | 429 Too Many | "잠시 후 다시 시도해주세요" |
| 서버 오류 | 500 Internal | "오류가 발생했습니다. 다시 시도해주세요" |
| 초대코드 무효 | 400 Bad Request | "유효하지 않은 초대 링크입니다" |

## 5. 로딩 & 빈 상태 (Loading & Empty States)

### 로딩 상태

```
폼 제출 중:
- "지금 예약하기" 버튼 → 로딩 스피너
- 버튼 텍스트: "예약 중..."
- 버튼 비활성화

예약 완료 페이지 로드 중:
- 스켈레톤 UI 표시 (순번, 초대 카드)
```

### 빈 상태

```
예약자 대시보드 (초대자 없음):
- "아직 초대한 친구가 없습니다"
- "초대 링크를 복사하여 친구에게 공유해보세요" (CTA)

대시보드 (데이터 로드 중):
- 지표 카드: 스켈레톤 로더
- 차트: 로딩 메시지
```
