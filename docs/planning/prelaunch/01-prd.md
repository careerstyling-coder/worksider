# 01-prd.md: 사전 예약 페이지 제품 요구사항 명세

**프로젝트**: Workside 사전 예약 페이지 (프리론칭)
**버전**: 1.0
**작성일**: 2026-03-25
**상태**: 사전 기획 완료

---

## MVP 캡슐 (10줄)

500명의 얼리어답터 직장인을 모으는 4주 사전 예약 캠페인. 공감 메시지로 시작해 Work Style DNA 소개, 이메일+직군+연차 예약폼 통해 관심사 포착. 예약 완료 후 고유 초대 링크 발급. 5명 초대 성공 시 얼리어답터 배지+풀 진단 우선 접근. 초대 전환율을 노스스타 지표로 삼아 바이럴 루프 검증. 2주 사전예약+2주 바이럴. 성공 조건: 500명 목표, 초대 전환율 20% 이상. 실패 신호: 2주차 초대 전환율 5% 미만 시 메시지/리워드 A/B 테스트.

## 1. 제품 개요

### 1.1 한 줄 설명
자신의 업무 스타일을 먼저 확인하고 싶은 직장인들을 위한 사전 예약 랜딩 페이지

### 1.2 Cogito Statement (창업자의 의도)
나는 Workside의 첫 사용자들과 관계를 맺고 그들의 행동(등록, 공유)을 통해 서비스의 가능성을 검증하고 싶다. 왜냐하면 10년간의 경험에서 사람과의 관계가 모든 것의 시작임을 알고, 성공적인 서비스들이 초기 사용자를 중요하게 다루는 것을 보아왔기 때문이다.

### 1.3 기간 & 목표
- **총 기간**: 4주 (2주 사전예약 + 2주 바이럴 확대)
- **예약 목표**: 500명 (얼리어답터 직장인)
- **초대 전환율 목표**: 20% 이상 (공유 링크 → 실제 예약)
- **배포 도메인**: workside.day (기존 서비스 기본 도메인)

## 3. 핵심 기능 (3개)

### FEAT-1: 사전 예약 랜딩 페이지
**목표**: 공감 메시지로 신뢰 형성, 기본 정보 수집

**기능 명세**:
- **공감 섹션 (HeroSection)**
  - 제목: "당신의 일 스타일을 먼저 알아보세요"
  - 부제: 10년 경험 기반 코칭 / 첫 500명을 위한 특별한 기회
  - 배경: 따뜻한 그래디언트 (신뢰감)

- **DNA 소개 섹션 (DNAIntroSection)**
  - 4축 설명 (P·C·Pol·S)
  - 각 축별 1~2줄 설명 + 아이콘
  - 페르소나 6가지 예시 (실무형 전문가, 전략적 성과자 등)

- **예약 폼 (ReservationForm)**
  - 필드: 이메일 (필수) / 직군 드롭다운 (필수) / 연차 드롭다운 (필수)
  - 직군 옵션: IT, 금융, 제조, 교육, 커뮤니케이션, 마케팅, 기타
  - 연차 옵션: 1년차, 2~3년차, 4~5년차, 6~10년차, 10년 이상
  - 이메일 중복 검증 (이미 예약한 경우 메시지)
  - 제출 버튼: "지금 예약하기"

**왜?**: 관심 있는 사람을 모으는 첫 접점. 공감으로 시작해 신뢰를 형성. 직군/연차로 타겟 분석 가능.

**우선순위**: MVP 필수

**완료 조건**:
- 폼 제출 성공 (이메일 중복 검증 포함)
- 모바일/데스크톱 반응형 100% 작동
- 로딩 시간 < 2초
- 예약 완료 후 `/reserved` 페이지로 자동 이동

---

### FEAT-2: 초대 바이럴 시스템
**목표**: 자발적 공유로 사용자 획득, 초대 전환율 측정

**기능 명세**:
- **초대 링크 발급**
  - 예약 완료 시 고유 초대 코드 생성 (nanoid 21자)
  - 링크 형식: `workside.day/?ref={invite_code}`
  - 복사 버튼 (클립보드 복사)

- **초대 진행바 (InviteProgressBar)**
  - "5명 초대하면 얼리어답터 배지 획득"
  - 시각적 진행바 (0/5 → 5/5)
  - 실시간 업데이트 (초대 성공할 때마다)

- **리워드 안내 (RewardInfo)**
  - 얼리어답터 배지 (프로필 표시)
  - 풀 진단 우선 접근 (4주 후 오픈 시 즉시 이용)

- **SNS 공유 버튼 (SocialShareButtons)**
  - 카카오톡 (커스텀 메시지)
  - 트위터 (OG 이미지)
  - 인스타그램 (링크 복사)

- **초대 랜딩 페이지 개인화**
  - 일반 방문 vs 초대 방문 구분 (`?ref={code}`)
  - 초대 방문 시 상단에 "OO님이 초대했어요" 배너
  - 나머지는 동일한 랜딩 페이지

**왜?**: 노스스타 지표(초대 전환율) 달성의 핵심 메커니즘. 자발적 행동이 진짜 검증. MBTI처럼 "나도 공유하고 싶은" 심리 활용.

**우선순위**: MVP 필수

**완료 조건**:
- 초대 링크 생성 & 복사 작동
- 초대 진행바 실시간 업데이트
- SNS 공유 클릭 시 올바른 링크로 이동
- `?ref={code}` 파라미터로 초대 인식 및 통계 저장

---

### FEAT-3: 행동 추적 대시보드
**목표**: 실시간 데이터로 바이럴 효과 검증, 관리자 의사결정 지원

**기능 명세**:
- **주요 지표 카드**
  - 총 예약 수
  - 초대 전환율 (%) = (초대로 인한 신규 예약) / (초대 링크 클릭)
  - 평균 초대자당 초대 성공 수
  - 얼리어답터 배지 획득자 수

- **차트**
  - 일별 예약 추이 (꺾은선 차트)
  - 직군별 분포 (원형/막대 차트)
  - 연차별 분포 (막대 차트)

- **랭킹 테이블**
  - 상위 초대자 (Top 10)
  - 각 초대자의 성공 초대 수

- **필터**
  - 기간 선택 (오늘/이번주/이번달/전체)

**왜?**: 행동 데이터로 서비스 가능성을 검증하는 핵심 도구. A/B 테스트 시 지표 변화 추적.

**우선순위**: MVP 필수 (기본 버전)

**완료 조건**:
- 실시간 데이터 갱신 (5분 주기)
- 모든 지표 계산 정확도 100%
- 모바일 대시보드도 열람 가능 (읽기 전용)
- CSV 내보내기 (선택, Phase 1.5)

## 5. 사용자 흐름 (4가지)

### Flow 1: 신규 방문 → 예약
```
홈페이지 접속 → 공감 메시지 확인 → DNA 소개 읽음
→ 이메일+직군+연차 입력 → 예약 완료
```

### Flow 2: 예약 → 초대
```
예약 완료 화면 도착 → 초대 링크 확인
→ SNS 공유 버튼 클릭 또는 링크 복사
→ 친구/동료에게 공유
```

### Flow 3: 초대 링크 수신 → 예약
```
초대 링크 방문 (workside.day/?ref=xxx)
→ "OO님이 초대했어요" 배너 확인
→ 공감 메시지 읽음 → 예약 폼 입력 → 예약 완료
→ 초대자의 진행바 업데이트 (1/5)
```

### Flow 4: 재방문 → 현황 확인
```
workside.day 재접속 → 로그인
→ 예약자 대시보드 도착
→ 내 순번, 초대 현황, 리워드 상태 확인
→ 추가 초대 링크 공유
```

## 8. 성공 지표 (KPI)

### North Star Metric
**초대 전환율** = (초대로 인한 신규 예약) / (초대 링크 클릭 수)
- Phase 1 목표: 20% 이상

### 핵심 지표

| 지표 | 목표 (4주 후) | 측정 방식 |
|-----|----------------|----------|
| 총 예약 수 | 500명 | Supabase 쿼리 |
| 초대 전환율 | 20%+ | (초대 신규) / (링크 클릭) |
| 평균 초대자당 성공 초대 | 2.5명 | (총 초대로 인한 신규) / (초대한 사람 수) |
| 직군 다양성 | 8개 직군 이상 | 산업군 분포 |
| 얼리어답터 배지 획득자 | 100명 이상 | (5명 이상 초대한 사람) |
| 일일 활성 사용자 (DAU) | 50+ | 로그인 또는 초대 링크 클릭 |

### 게이트 기준 (사전예약 → 본 서비스)
1. 총 예약 500명 도달
2. 초대 전환율 15% 이상 (또는 2주차 데이터 기반 A/B 테스트)
3. 직군 다양성 6개 이상 (표본의 대표성)

## 9. 가설 vs 사실 (Assumption vs Validated)

### 가설 (미검증)

| ID | 가설 | 검증 방법 | 기대값 | 우선순위 |
|----|------|---------|--------|---------|
| H-1 | 직장인이 "업무 스타일" 진단에 관심이 있다 | 1주일 내 예약 수 > 100 | 500명 in 2주 | HIGH |
| H-2 | 초대 메커니즘이 바이럴 루프를 작동시킨다 | 초대 전환율 > 15% | 20% 목표 | CRITICAL |
| H-3 | SNS 공유가 주요 획득 채널이다 | 공유 채널별 유입 > 30% | 2주 내 확인 | HIGH |
| H-4 | 리워드(배지+우선 접근)가 초대 동기를 높인다 | 리워드 설명 후 초대율 상승 | 15% 이상 | MEDIUM |
| H-5 | 초대 랜딩 페이지의 개인화 배너가 전환율을 높인다 | 개인화 vs 비개인화 A/B 테스트 | 개인화 +10% | MEDIUM |

### 사실 (검증됨)
- MVP Phase 1 (본 서비스) 구현 가능
- 초대 바이럴 시스템 기술적 구현 가능 (Supabase + Next.js)

## 10. 벤치마킹 (유사 사전예약 서비스)

### Case 1: Notion 초기 사전예약 (2020)
| 항목 | Notion | 우리 학습점 |
|------|--------|-----------|
| 길이 | 수개월 대기 | 우리는 4주 단기 집중 |
| 전환율 | 공개 안 됨 (추정 10~15%) | 초대 전환율 20% 목표로 상향 |
| 바이럴 메커니즘 | 친구 초대 시 대기열 앞당김 | 우리도 유사 구조 (초대 배지+우선) |

### Case 2: Superhuman 대기열 시스템
| 항목 | Superhuman | 우리 학습점 |
|------|-----------|-----------|
| UX | "순번" 시각화 (심리학) | 우리도 "/reserved"에서 순번 표시 |
| 공유 동기 | 순번 앞당김 + 초대 카운트 | 우리도 초대 진행바로 가시화 |
| 성과 | 초기 1주일 5천명 (강함) | 500명 목표는 더 리얼한 규모 |

### Case 3: Clubhouse 초대제 (2020-2021)
| 항목 | Clubhouse | 우리 학습점 |
|------|-----------|-----------|
| 초대 가치 | 높은 희소성 (iOS만, 초대만) | 우리는 희소성 약함 → 리워드로 보충 |
| 바이럴 속도 | 급격한 성장 후 추락 | 우리는 지속 가능성 중심 (관계 기반) |
| 실패 원인 | 실질 가치 부족 | DNA 진단 자체가 가치 → 차별화 |

### Workside의 위치
**"직장인 맥락 + 관계 기반" = 차별화 포인트**
- Notion/Superhuman: 제품 희소성이 초대 동기 (우리는 약함)
- Clubhouse: 바이럴만 중시 (우리는 관계 중시)
- **Workside: 초대자-피초대자 관계 + 공통 언어(DNA) = 커뮤니티의 시작**

## 11. MVP 범위 정의

### 포함 사항 (Phase 1: 사전예약)
- 사전 예약 랜딩 페이지 (공감+DNA소개+폼)
- 예약 완료 화면 + 초대 링크 발급
- SNS 공유 버튼 (카카오톡, 트위터, 인스타)
- OG 이미지 동적 생성 (`@vercel/og`)
- 초대 랜딩 페이지 개인화 배너
- 예약자 대시보드 (내 순번, 초대 현황)
- 관리자 대시보드 탭 추가 (예약 통계)

### 제외 사항 (4주 후, Phase 2: 본 서비스)

- 다크 모드
- 국제화 (영어)

## 12. 리스크 & 완화 전략

| 리스크 | 신호 | 영향 | 완화 전략 |
|--------|------|------|---------|
| 초대 전환율 너무 낮음 | 2주차 전환율 < 10% | 바이럴 실패 → 500명 미달 | 1주차 A/B 테스트: 배너 메시지/리워드 변형 테스트 |
| 첫주 예약 수 저조 | 1주일 < 50명 | 캠페인 구성 문제 | 초기 타겟팅(SNS, 이메일) 강화, 주변 네트워크 우선 |
| 이메일 중복 가입 (스팸) | 일일 중복율 > 20% | 데이터 품질 악화 | 이메일 인증 (OTP) 추가, IP 기반 rate limiting |
| 사전 예약 중도 포기 | 폼 입력 후 미제출 > 30% | 전환율 저하 | 폼 단순화 (필드 3개 → 2개 검토), 모바일 UX 최적화 |
| 초대 시스템 버그 | 초대 진행바 미업데이트 | 사용자 신뢰도 하락 | 실시간 데이터 정확성 테스트, 대시보드 모니터링 (5분마다) |
| SNS 공유 저조 | 공유 버튼 클릭 < 5건/일 | 유입 채널 약화 | OG 이미지 최적화, 공유 문구 개선, 인센티브 강화 |

## 13. 타협 기록 (Tradeoff)

| 기능 | 결정 | 이유 |

| 모바일 앱 | 웹 반응형으로 충분 | MVP 범위 축소 |
