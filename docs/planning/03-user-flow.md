# 03-user-flow.md: 사용자 플로우

**프로젝트**: Workside (워크사이드)
**버전**: 1.0
**작성일**: 2026-03-16
**상태**: MVP Phase 1 기획

---

## 1. 플로우 개요

Workside의 사용자 여정은 4가지 주요 경로로 나뉜다. 각 플로우는 시작점이 다르지만, 최종 목표는 '정기적 질문 참여'를 통한 커뮤니티 재방문 유도이다.

---

## 2. Flow 1: 공유 링크 수신 플로우 (Shared Link)

**시나리오**: 친구가 자신의 DNA 결과를 SNS나 카카오톡으로 공유했고, 수신자가 링크를 클릭

```
START (클릭: 공유 링크)
  ↓
[공유 수신 화면] (/share/:shareId)
  ├─ 공유자의 DNA 결과 표시 (이름 없음, 점수만)
  │  "P: 72, C: 45, Pol: 38, S: 61 → 전략적 성과자"
  └─ "나는 어떨까?" CTA 버튼
  ↓
[DNA 진단] (/diagnosis)
  ├─ 버전 선택: 세미 (3~5분) / 풀 (15~20분)
  ├─ 문항 진행 (진행률 표시)
  └─ 진단 완료
  ↓
[DNA 결과] (/result/:sessionId)
  ├─ 레이더 차트 + 4축 점수
  ├─ 페르소나 설명
  └─ SNS/카카오/링크 공유 버튼
  ↓
[가입 게이트] ← ⭐️ 여기서 가입 강제 (모듈화 가능)
  ├─ 이메일 + 닉네임 + 비밀번호
  ├─ 산업군 선택 (드롭다운)
  ├─ 회사 규모 선택 (카드형)
  └─ "가입하기" 버튼
  ↓
[피드] (/feed)
  ├─ 진행 중인 질문 목록
  ├─ 첫 번째 질문 자동 노출
  └─ "참여하기" CTA
  ↓
[질문 참여] (/question/:questionId)
  ├─ 질문 읽기
  ├─ 선택지 선택
  └─ "제출" 버튼
  ↓
[세부 결과] (/question/:questionId/result)
  ├─ 전체 집계 (%)
  ├─ 유형별 분포 바 차트
  ├─ 내 유형 하이라이트
  ├─ 소수 의견 (Minority View)
  └─ "다른 질문 보기" 또는 "제안하기"
  ↓
END (정기적 재방문 또는 이탈)
```

**주요 전환점 (Conversion Points)**:
1. 공유 수신 → 진단 시작: 호기심 (나는 어떨까?)
2. 진단 완료 → 결과 조회: 즉시 피드백
3. 결과 조회 → 가입: 결과 저장 욕구
4. 가입 → 첫 질문 참여: 관성 (이미 여기 있으니)
5. 질문 참여 → 재방문: 결과 확인 욕구

## 3. Flow 2: 직접 방문 플로우 (Organic/Direct)

**시나리오**: 검색(Google)이나 직접 URL 입력으로 랜딩페이지 방문

```
START (검색 또는 직접 URL: workside.co)
  ↓
[랜딩 페이지] (/)
  ├─ 서비스 소개 영상/이미지
  ├─ "DNA 진단 시작" CTA (가장 큰 버튼)
  ├─ 피처 소개
  ├─ 사용자 후기/결과
  ├─ 로그인 링크 (우상단)
  └─ "가입하기" 링크 (우상단)
  ↓
[DNA 진단] (/diagnosis)
  ├─ 버전 선택: 세미 / 풀
  ├─ 문항 진행
  └─ 진단 완료
  ↓
[DNA 결과] (/result/:sessionId)
  ├─ 레이더 차트
  ├─ 축별 설명
  └─ 공유 버튼 (→ 바이럴 루프)
  ↓
[가입 게이트]
  ├─ 이메일 + 비밀번호 입력
  ├─ 산업군 + 회사 규모
  └─ 가입 완료
  ↓
[피드] (/feed)
  └─ 첫 질문 참여
  ↓
[세부 결과]
  └─ 다른 질문 또는 제안
  ↓
END (정기적 재방문)
```

**특징**:
- Flow 1 (공유)과 동일하지만, 진입점이 랜딩페이지
- 동기 부족 가능성 높음 (친구의 비교 자극 없음)
- 랜딩페이지의 설득력이 핵심

## 4. Flow 3: 회원 재방문 플로우 (Returning User)

**시나리오**: 이미 가입한 사용자가 로그인해서 재방문

```
START (로그인 또는 자동 세션)
  ↓
[피드] (/feed) ← 기본 목적지
  ├─ 진행 중인 질문 목록 (상단에 채택된 질문 고정)
  ├─ 필터: 진행중 / 마감 / 내 참여 상태
  ├─ 제안 목록 (최신순 또는 Shout out 순)
  └─ "질문 제안" CTA
  ↓
  ┌─────────────────┬─────────────────┬──────────────────┐
  ↓                 ↓                 ↓                  ↓
[질문 참여]    [내 DNA 보기]    [질문 제안]      [마감된 질문]
(/question/:id)  (/my-dna)       (/suggest)      (결과만 열람)
  │              │                 │                 │
  ├─ 질문 읽기   ├─ 진단 요약      ├─ 제안 폼       └─ 결과 집계
  ├─ 선택지      ├─ 참여 이력      ├─ 주제 설명      └─ 유형별
  ├─ 제출        ├─ 재진단 버튼    ├─ 배경 설명      분포 보기
  └─ 결과 확인   ├─ 공유 버튼      ├─ 제출
                 └─ 산업/규모 태그 └─ 검토 대기
                                   메시지
  ↓                 ↓                 ↓
[세부 결과]   (프로필 피드 내)  (제안 목록)
(/question/:id/result)
  ├─ 집계 %
  ├─ 유형별 분포
  ├─ 소수 의견
  └─ Shout out
      on 제안
  ↓
[피드] ← 루프
```

**특징**:
- **메인 진입점**: /feed
- **스프레드**: 질문 참여, 내 DNA 조회, 질문 제안, 마감된 질문 결과 등으로 분산
- **참여 메커니즘**: 계속 새로운 질문이 추가되므로 재방문 동기 유지
- **커뮤니티 참여**: Shout out으로 제안 우선순위 영향

## 5. Flow 4: 알림/직접 이동 플로우 (Notification)

**시나리오**: 이메일 알림을 받고 특정 질문으로 직접 이동

```
START (이메일 클릭)
  │
  └─ "새로운 질문이 올라왔습니다" 링크
  ↓
[질문 상세] (/question/:questionId)
  ├─ 질문 제목 + 설명
  ├─ 선택지 표시
  └─ "답변하기" 버튼
  ↓
[질문 참여]
  ├─ 선택지 선택
  └─ 제출
  ↓
[세부 결과]
  ├─ 결과 확인
  └─ "다른 질문 보기" (→ 피드)
  ↓
[피드]
  └─ 다른 활동 계속
  ↓
END
```

**특징**:
- 가장 짧은 경로
- 알림 타이밍이 핵심 (너무 잦으면 스팸, 너무 적으면 효과 없음)
- Phase 2에서 구현 (Realtime 또는 scheduled email)

## 6. 텍스트 기반 사용자 여정 맵

```
비회원 (Anonymous)
├─ 공유 링크 수신
│  └─ 진단 → 결과 → 가입 → 피드 (High Conversion)
│
└─ 직접 방문
   └─ 랜딩 → 진단 → 결과 → 가입 → 피드 (Medium Conversion)

회원 (Authenticated)
├─ 로그인
│  └─ 피드 (메인 디스플레이)
│     ├─ 진행중 질문 참여 (Primary Action)
│     ├─ 마감된 질문 결과 조회 (Secondary)
│     ├─ 내 DNA 재조회 (Profile)
│     └─ 질문 제안 (Create)
│
└─ 이메일 알림
   └─ 특정 질문 → 직접 참여 → 결과 → 피드 (Re-engagement)

관리자 (Admin)
└─ /admin (대시보드)
   ├─ 지표 모니터링
   ├─ 질문 등록/배포
   ├─ 제안 채택/반려
   └─ 사용자 분석
```

## 7. 게이트웨이 & 결정 포인트

### 7.1 가입 게이트 (Signup Gate)

**위치 옵션** (관리자 설정으로 변경 가능):
1. **결과 후 즉시** (권장): `/result/:sessionId` 후 `/signup`
   - 장점: 높은 전환율 (결과 호기심으로 강한 동기)
   - 단점: 비회원 데이터 손실

2. **피드 입장 전**: `/feed` 접근 시 `/signup`
   - 장점: 첫 질문 참여 전 확보
   - 단점: 중간 이탈 가능성

3. **질문 결과 조회 시**: `/question/:id/result` 접근 시 `/signup`
   - 장점: 2회차 참여 시점에서 확보
   - 단점: 1회차 비회원 데이터 수집 후 전환

**구현**:
```typescript
// adminSettings에서 동적 변경
const gatePosition = await getAdminSetting('signup_gate_position');

if (gatePosition === 'after_result' && !isAuthenticated) {
  redirect('/signup?from=result');
}
```

### 7.2 비회원 vs 회원 분기

| 행동 | 비회원 | 회원 |
|------|--------|------|
| 진단 | ✅ 가능 | ✅ 가능 (저장됨) |
| 결과 열람 | ✅ 가능 | ✅ 가능 (히스토리) |
| 질문 참여 | ✅ 가능 (익명) | ✅ 가능 (기록됨) |
| 결과 열람 | ❌ 불가 (잠금) | ✅ 가능 |
| 제안 | ❌ 불가 | ✅ 가능 |
| Shout out | ❌ 불가 | ✅ 가능 |

## 8. 재방문 루프 분석

### 8.1 1차 참여 → 2차 재참여 (핵심 메트릭)

```
Day 0 (사용자 획득)
├─ DNA 진단 완료
├─ 결과 조회
└─ 가입

Day 1~3 (초기 참여)
├─ 질문 참여 (1회차)
└─ 결과 조회

Day 4~7 (재참여 기회)
├─ 새로운 질문 배포 (알림 또는 피드)
├─ 질문 참여 (2회차) ← **핵심 신호**
└─ 결과 조회 + 이전 결과와 비교

Day 8~14+ (습관화)
├─ 정기적 재방문
├─ 질문 제안 활동
└─ 커뮤니티 상호작용 (Shout out)
```

### 8.2 이탈 포인트 (Risk)

| 포인트 | 이탈 신호 | 대응 |
|--------|---------|------|
| 진단 → 결과 | 느린 로딩, 충격적 결과 | UX 최적화, 긍정적 프레이밍 |
| 결과 → 가입 | 명확한 가입 필요성 부재 | "결과 저장", "질문 참여" CTA 강화 |
| 가입 → 첫 질문 | 질문 난도 높음, 모호함 | 쉬운 질문으로 시작, 명확한 선택지 |
| 1회 참여 → 재참여 | 새로운 질문 없음, 알림 부족 | 정기 배포 (주 2회+), 이메일 알림 |

## 9. 사용자 세그먼트별 경로

### 9.1 '빠르게 확인하고 싶은' 사용자
```
공유 링크 → [세미 진단 5분] → 결과 → 공유 → 가입 (낮음) → 이탈
```
**대응**: 세미 결과로 충분하다는 메시지, 풀 진단 "나중에" 옵션

### 9.2 '깊이 있게 탐색하려는' 사용자
```
랜딩 → [풀 진단 20분] → 상세 결과 → 가입 (높음) → 질문 정기 참여
```
**대응**: 풀 진단 강조, 심화 분석 프리뷰, 코칭 세션 언급

### 9.3 '커뮤니티 활동을 좋아하는' 사용자
```
가입 → 피드 → [질문 다다다 참여] → 제안 → Shout out → 성장 참여자화
```
**대응**: 피드 우선 노출, 제안 쉽게 하기, 상호작용 기능 강화

## 10. 플로우 구현 체크리스트

### Phase 1 (MVP)
- [ ] Flow 1 (공유): 공유 링크 → 진단 → 결과 → 가입 → 피드
- [ ] Flow 2 (직접): 랜딩 → 진단 → 결과 → 가입 → 피드
- [ ] Flow 3 (재방문): 로그인 → 피드 → 질문 → 결과
- [ ] 가입 게이트 (모듈화)
- [ ] 비회원 익명 응답 + 결과 잠금

### Phase 2 (확장)
- [ ] Flow 4 (알림): 이메일 → 질문 직접 이동
- [ ] 실시간 알림 (Supabase Realtime)
- [ ] 비교 기능 (공유자 vs 수신자)
- [ ] 고급 세그먼테이션
