### S-2: 공유 수신 화면 (/share/:shareId)

**경로**: `/share/:shareId`
**접근**: 퍼블릭 (공유 링크 수신자)
**목적**: 공유자의 DNA 결과 열람 및 진단 시작

**데이터 흐름**:
1. URL에서 `shareId` 추출
2. Supabase에서 dna_results 테이블 조회 (share_token = shareId)
3. 점수 + 페르소나 렌더링
4. "나는 어떨까?" 클릭 시 진단 시작

**주요 구성**:

```
┌─────────────────────────────────────────┐
│ Header (간단함)                         │
│ ├─ 로고 (작음)                          │
│ └─ [닫기] 또는 X                        │
├─────────────────────────────────────────┤
│ Shared Result Card                      │
│ ├─ 제목: "친구가 공유한 Work DNA"       │
│ ├─ RadarChart (비활성)                  │
│ │  P: 72, C: 45, Pol: 38, S: 61       │
│ ├─ 페르소나: "전략적 성과자"           │
│ └─ 페르소나 설명 텍스트 (2~3줄)       │
├─────────────────────────────────────────┤
│ Call-to-Action                          │
│ ├─ 큰 텍스트: "나는 어떨까?"           │
│ ├─ 부설명: "같은 방식으로 진단받기"    │
│ └─ [진단 시작] Primary 버튼            │
├─────────────────────────────────────────┤
│ 로그인 제안 (선택)                      │
│ └─ "결과를 저장하려면 가입하세요"      │
└─────────────────────────────────────────┘
```

**주요 기능**:
- SSR로 OG 이미지 동적 생성
- 공유자 결과 표시 (이름 없음, 점수만)
- 페르소나별 설명 텍스트
- "진단 시작" 클릭 → `/diagnosis` (공유 정보 전달)

**컴포넌트**:
- ShareResultCard (레이더 차트 + 페르소나)
- PersonaDescription (설명)
- CTASection (버튼)

### S-3: DNA 진단 (/diagnosis)

**경로**: `/diagnosis`
**접근**: 퍼블릭 (비회원/회원 모두)
**목적**: DNA 진단 설문 진행

**플로우**:

```
1. 버전 선택 화면
   ├─ [세미 버전 (12문항, 3~5분)] 카드
   └─ [풀 버전 (40문항, 15~20분)] 카드

2. 진단 진행 화면
   ├─ 진행률 바 (예: 5/12)
   ├─ 질문 번호 (1/12)
   ├─ 질문 텍스트
   ├─ 7점 Likert 척도 (라디오 버튼 또는 슬라이더)
   │  ① 강력히 동의 안함 ← → ⑦ 강력히 동의
   ├─ [이전] [다음] 버튼
   └─ [저장 및 나중에 계속] (선택)

3. 완료 화면
   └─ 로딩 + "결과 계산 중..."
```

**주요 기능**:
- 버전 선택 (세미 / 풀)
- 문항별 진행 상황 표시
- 7점 Likert 척도 입력 (반응형 슬라이더 모바일용)
- 이전/다음 네비게이션
- 진단 세션 자동 저장 (로컬 스토리지 또는 DB)
- 완료 후 `/result/:sessionId`로 자동 리다이렉트

**상태 관리**:
```typescript
interface DiagnosisState {
  sessionId: string;
  version: 'semi' | 'full';
  responses: { questionId: string; value: number }[];
  currentQuestionIndex: number;
  isCompleting: boolean;
}
```

**컴포넌트**:
- VersionSelector (카드형 선택)
- ProgressBar (진행률)
- QuestionCard (질문 + 척도)
- LikertScale (라디오 또는 슬라이더)
- NavigationButtons

### S-5: 가입 플로우 (/signup)

**경로**: `/signup`
**접근**: 퍼블릭 (가입 게이트에서 유입)
**목적**: 이메일/비밀번호 입력 + 온보딩 정보 수집

**플로우**:

```
Step 1: 기본 정보
├─ 이메일 입력 (email 검증)
├─ 닉네임 입력 (2~20자)
├─ 비밀번호 입력 (8자 이상, 특수문자 권장)
├─ 비밀번호 확인
└─ [다음] 버튼

Step 2: 추가 정보 (선택적)
├─ 산업군 선택 (드롭다운)
│  ├─ IT/소프트웨어
│  ├─ 금융
│  ├─ 제조
│  ├─ 서비스
│  ├─ 기타
│  └─ 선택하지 않음
├─ 회사 규모 (카드형 선택)
│  ├─ 스타트업 (S)
│  ├─ 중소기업 (M)
│  ├─ 중견기업 (L)
│  └─ 대기업 (XL)
└─ [가입 완료] Primary 버튼

Step 3: 완료 화면
├─ "가입되었습니다!"
└─ [피드로 이동] 또는 자동 리다이렉트
```

**주요 기능**:
- 이메일 중복 확인 (실시간)
- 비밀번호 강도 표시기
- Supabase Auth와 통합
- 산업군/회사규모 선택사항
- 진행률 표시 (Step 1/2)
- 입력 검증 + 오류 메시지

**컴포넌트**:
- SignupForm (다단계)
- InputField (이메일, 닉네임, 비밀번호)
- IndustrySelector (드롭다운)
- CompanySizeSelector (카드형)
- ProgressIndicator

## 5. 모바일 vs 데스크톱 차이

| 요소 | 모바일 | 데스크톱 |
|------|--------|---------|
| **헤더** | 버거 메뉴 | 가로 네비게이션 |
| **버튼 크기** | 48px (높이, 터치) | 40px |
| **카드 배치** | 1 컬럼 (100%) | 2~3 컬럼 |
| **차트** | 100% 너비, 작은 크기 | 고정 너비 (400px) |
| **테이블** | 카드 형식 (세로) | 가로 테이블 |
