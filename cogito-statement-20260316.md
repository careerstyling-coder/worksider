## Phase 2: 코기토 명제

### 코기토 3문 응답

| 질문 | 응답 |
|------|------|
| Q1: 순수 의도 | 일에서의 '다름'을 겉으로 꺼내어 말할 수 있는 언어를 만들고, 함께 일하는 법을 익히는 문화를 만들고 싶다 |
| Q2: 자생성 | 내면에서 자생. 10년간 코칭 현장에서 "사람을 이해하는 것은 불가능하지만, 일에 대한 태도의 다름은 볼 수 있고, 그것이 보이면 함께 일하는 법을 찾을 수 있다"는 통찰을 반복 확인. 사업의 본질은 서로가 함께 뭉쳐 안되던 일을 되게 하는 것이고, 그 출발은 다름을 인정하는 것 |
| Q3: 절실함 | 완성되면 국내를 넘어 전 세계에 퍼뜨릴 방법을 고민하게 된다. 삶이 근본적으로 달라지는 수준의 절실함 |

### 코기토 명제 (한 문장)

> **"나는 일에 대한 태도의 '다름'을 겉으로 꺼내는 언어를 만들고 싶다,
>  왜냐하면 사람을 이해하는 것은 불가능하지만 일에 대한 태도의 다름은 볼 수 있고,
>  그것이 보이면 함께 일하는 법을 찾을 수 있기 때문이다.
>  이것이 실현되면, 전 세계 직장인에게 퍼뜨릴 방법을 고민하게 된다."**

## Phase 3: 재건 (Codo)

### 연역 계층

| Layer | 내용 | 코기토 연결 |
|-------|------|------------|
| 0. 의도 | 일에 대한 태도의 '다름'을 겉으로 꺼내는 언어를 만든다 | 토대 |
| 1. 문제 | 직장인들이 자신과 타인의 일 태도를 표현할 공통 언어가 없어서, 다름을 인식하지 못하고, 마찰의 원인을 설명할 수 없다 | <- Layer 0에서 도출 |
| 2. 기능 | DNA 진단(공통 언어 생성) + 일 질문 참여(다름 경험) + 유형별/소수 의견 비교(다름 인식) | <- Layer 1에서 도출 |
| 3. 기술 | 점수 계산 엔진 + 레이더 차트 + 익명 응답 집계 + DNA↔응답 연결 + OG 이미지 동적 생성 + 이메일 인증 | <- Layer 2에서 도출 |
| 4. 구조 | SSR+CSR 풀스택 웹앱 + 관계형 DB + 기능 플래그 ON/OFF + 모바일 반응형 웹 | <- Layer 3에서 도출 |
| 5. 도구 | Next.js + Supabase(PostgreSQL) + Chart.js/Recharts + Vercel + Tailwind CSS | <- Layer 4에서 도출 |

### 연역 추적 (Deduction Trail)

```
Layer 0 → 1: "태도의 다름을 꺼내는 언어를 만들고 싶다"이므로,
             문제는 "일 태도를 표현할 공통 언어가 없다"이다.
             논리: 언어가 없으면 다름을 꺼낼 수 없으므로, 언어 부재가 핵심 문제.

Layer 1 → 2: "공통 언어 부재"를 풀려면,
             ① 개인의 태도를 구조화하는 진단(언어 생성)
             ② 같은 질문에 다른 답을 보는 경험(다름 인식)
             ③ 유형별 비교(다름의 구체적 확인)이 필요하다.
             제외: 질문 제안(없어도 문제 해결 가능 — 관리자가 등록)
             논리: 언어 생성 → 언어 사용 경험 → 언어로 다름 확인 순서.

Layer 2 → 3: "진단 + 참여 + 비교"를 구현하려면,
             점수 계산(구조화) + 차트(시각화) + 익명 집계(다수의 답) +
             프로필↔응답 연결(유형별 분류) + 공유(확산)가 필요하다.
             논리: 구조화 → 시각화 → 집계 → 분류 → 확산.

Layer 3 → 4: "점수 + 차트 + 집계 + 인증"을 실현할 구조는
             SSR+CSR 풀스택 웹앱 + 관계형 DB이다.
             대안: 순수 SPA → SEO 불가(바이럴 진입 불리) → 폐기.
             대안: NoSQL → 유형별 집계 쿼리에 불리 → 폐기.
             논리: 서버 필수(인증+집계) + SEO 필요(바이럴) → SSR 하이브리드.

Layer 4 → 5: "SSR+CSR + PostgreSQL"을 구현할 도구는
             Next.js + Supabase + Vercel이다.
             논리: SSR 생태계 최적 + 인증 내장 + 글로벌 CDN(코기토: 전 세계에 퍼뜨린다).
```

### 코기토 역추적 검증

```
Next.js+Supabase(L5) ← SSR+CSR+PostgreSQL(L4) ← 점수엔진+차트+집계(L3) ← 진단+참여+비교(L2) ← 공통언어부재(L1) ← 다름을꺼내는언어(L0)
→ 논리 연결: 확인 완료. 끊김 없음.
```
